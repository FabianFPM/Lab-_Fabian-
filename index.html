Bitácora 1

Introducción al curso

23 de Marzo de 2023 Jane Doe

En la primera sesión, tuvimos una breve introducción al curso, con una referencia a Steve Jobs. Luego, nos acercamos a HTML5, que es un lenguaje de marcado de hipertexto utilizado para crear sitios web y aplicaciones web. Está diseñado para mejorar la interactividad, funcionalidad y apariencia de los sitios web. HTML5 incluye etiquetas nuevas y mejoradas, así como nuevas características como el soporte para audio y video, una arquitectura de aplicación para la programación de aplicaciones web, y una API de geolocalización para facilitar la navegación. HTML5 también mejora la compatibilidad con dispositivos móviles y permite la creación de contenido dinámico para una mejor experiencia de usuario.

Día de expos

31 de Marzo de 2023 Jane Doe

Este día estuvimos la gran parte de la clase en exposiciones sobre problemáticas, y vimos las posibles soluciones, identificando su Objetivo de Desarrollo Sostenible. Una ODS es un tipo de estructura de datos en la que los datos se organizan de una manera lógica y estructurada para permitir una mejor búsqueda y recuperación. Bitácora 12

diagrama de flujo

28 de Abril de 2023 Jane Doe

En la clase, vimos la representación del diagrama de flujo, donde se hace más hincapié en Void Setup y Void Loop. También aprendimos opciones como delay, high y low para un funcionamiento óptimo de Arduino. Es importante tener en cuenta que Void Setup se utiliza para configurar la placa Arduino y que solo se ejecuta una vez al inicio del programa, mientras que Void Loop se utiliza para la ejecución continua del código. Además, se puede añadir una variedad de funciones dentro de Void Loop, como la lectura de sensores, la activación de actuadores o la comunicación con otros dispositivos.

Bitácora 14

diagram de flujo pt 2

05 de Mayo de 2023 Jane Doe

En la clase, vimos cómo graficar en el diagrama de flujo. En el caso del If, se utiliza un rombo que en el programa se representa con la palabra "si", y que tiene dos posibles salidas: una si se cumple la condición y otra si no se cumple. Es importante recordar que el "if" se utiliza para tomar decisiones en el programa, como, por ejemplo, si un valor es mayor o menor que otro, o si un sensor está activo o no. Con esta herramienta, podemos crear programas más complejos que nos permitan automatizar procesos o tomar decisiones en tiempo real. Además, en el diagrama de flujo también se pueden representar otras estructuras de control de flujo, como el "for" y el "while", que nos permiten repetir ciertas acciones un número determinado de veces o mientras se cumpla una condición.
